# dolbom studio — 설계 문서

> 돌봄로봇중개연구사업단 주간 업무·협업 웹앱 (브랜드명 **Dolbom Studio**)
> 배포: https://dolbom-studio.streamlit.app
> 작성/유지: 김건양 연구원 (Claude Code / Codex 협업) · 최종 갱신 2026-07-09

## 1. 한 줄 요약

**팀원이 각자 개인 계정으로 로그인해 쓰는, 돌봄로봇 사업단의 주간 업무·협업 웹앱.** 매주 반복되던
"10명 보고 → 한글 취합본 복붙"을 **버튼 한 번**으로 바꾸고, 구매·장비·방문·협업·공지·일정·뉴스를
한 화면에 모았다. 폰에서는 **홈 화면에 추가(PWA)** 로 앱처럼 쓴다.

## 2. 왜 시작했나 (문제 정의)

| 업무 | 기존 방식 | 문제 |
|------|-----------|------|
| 주간 업무보고 | 각자 한글/워드 작성 → 한 명이 **복붙 취합** | 취합자 매주 30~60분 수작업, 서식 깨짐, 미제출 파악 수동 |
| 공통확인사항 | 최혜민 연구원이 표 별도 관리 | 매주 한글 표 다시 제작 |
| 구매요청 | 엑셀 양식 개별 작성 | 양식 제각각, 수합 번거로움 |
| 장비 사용현황 | 엑셀 연구별 시트 수식 | 시트 동기화 부담 |
| 문제 접수/FAQ | 카톡·구두 | 기록·미해결 추적 안 됨 |
| 공지·일정 | 카톡 | 흘러가 놓침 |

→ **"입력은 각자, 취합·공유는 자동"** 하는 단일 웹앱으로 통합.

## 3. 목표 & 설계 원칙

- **취합 자동화 우선** — 사람이 하던 복붙을 없앤다(최대 페인포인트).
- **결과물은 기존 한글 양식 그대로**, 입력은 웹 폼으로 쉽게.
- **한 화면에서 파악** — 홈에 공지·할 일·일정·뉴스.
- **개인 계정 + 관리자 승인** — 누가 뭘 했는지 계정 기반(2026-07, 공용 비번 폐지).
- **데이터가 날아가지 않게** — 다층 백업(구글 복제 + 버전기록 + 휴지통 + 오프라인 백업).
- **폰 우선 접근성** — 반응형 + 홈 화면 추가(PWA).
- **점진적 확장** — 기능 1개 = 페이지 함수 + `*_store.py` 모듈.

## 5. 인증 — 개인 계정 로그인 (2026-07 전환)

공용 비밀번호(`carerobot`)를 폐지하고 **개인 계정 + 관리자 승인**으로 전환.

- 저장: 구글시트 `계정` 탭 — `[아이디, 비번(해시), 이름, 직함, 이메일_korea, 이메일_gmail, 상태, 가입일시, 승인일시]`
- **회원가입**: 이름·직함·아이디·비번(+확인)·이메일(korea·gmail) → 상태 `대기`.
- **관리자 승인**: `team_config.ADMIN_IDS`(현재 `{"admin"}`)에 있는 아이디로 가입하면 **자동 승인 + 승인 권한**.
  관리자는 사이드바 **👤 회원 관리**에서 전체 계정 상태(승인/대기/거부)를 변경(거부 복구·권한 회수 가능).
- **로그인**: 승인된 계정만. 로그인하면 `me`=이름, `is_admin` 세팅 → 모든 폼에 본인 이름 자동.
- **보안**: 비번은 **pbkdf2-sha256 해시**(평문 저장 X). 새로고침·자동 로그인은 **비번해시 기반 토큰**
  `?uid=&tok=`(위조 불가) + 선택적 **localStorage 저장**("저장된 계정으로 자동 로그인" 버튼).
  보안 리뷰 통과(암호·토큰·로그아웃 정상, 거부 복구·행밀림 방지 반영).

## 6. 기능 (사이드바 메뉴)

| 메뉴 | 하는 일 |
|------|---------|
| 🏠 홈 | 상단 워드마크 → 바로가기 타일 → 공지 → 좌(오늘 챙길 것·내 할 일)/우(7일 일정) → 달력 → 뉴스 |
| 📝 업무보고 작성·취합 | (탭)작성 / 제출현황·**HWPX 취합본 생성**·백업 / 공통확인사항 |
| 🏠 스마트돌봄스페이스 | FAQ 수집 / 관리대장 문제 접수·완료 |
| 🛒 구매요청서 | 품목 표 → 시트 누적 + 엑셀 양식 + 완료처리 |
| 📋 문서 협업 | 구글문서 링크 보드 + 제출현황 + (OAuth)파일 자동변환 |
| 🔧 장비 사용현황 / 📍 실증 방문 일지 / 📚 과거 회의록 열람 | 조회·편집·기록 |
| 👤 회원 관리 (관리자) | 가입 승인·상태 변경 |

- **홈 바로가기**: 순수 HTML 타일 그리드(네이버식 아이콘). 클릭=`?go=` 링크(로그인 유지, 메뉴 이동/토글).
  공지등록·일정 관리(＋)는 같은 방식 토글.
- 사이드바 **기본 접힘**(폰/새로고침 시 다크 사이드바 방해 없게). 메뉴는 `»`로 연다.

## 8. 모바일 / PWA

- **반응형**: `@media (max-width:700px)` — 좁은 화면에서 2단 컬럼을 세로로 쌓고 여백 축소, 워드마크 자동 축소.
- **PWA(홈 화면에 추가)**: 폰 브라우저로 접속 → "홈 화면에 추가" → DS 아이콘으로 앱처럼 전체화면 실행.
  네이티브 앱/스토어 등록 불필요·비용 0. (스토어가 필요해지면 웹뷰 래퍼로 확장 가능)
- 팀원 사용법은 `docs`(또는 별도 안내)로 배포.

## 9. 핵심 설계 결정 & 근거

- **HWPX 취합본 = 셀 채우기**(동적 행 삽입은 한글 파서가 깨져서, 검증된 리프 표에 고정좌표). `TROUBLESHOOTING.md`에 과거 범인 기록.
- **구글시트 = 저장소**(서비스계정 R/W). 화재로 데이터 손실 경험 → 4중 안전.
- **드라이브 파일은 OAuth 본인계정**(서비스계정은 드라이브 저장 불가).
- **개인 계정 전환**(2026-07): 공용 비번 폐지, 계정+관리자 승인. 비번 해시·토큰.
- **문서협업 자동 공지**(가상 렌더, 완료·삭제 시 소멸) · **공지 만료 자동삭제**(RAW 저장으로 날짜 비교 안정).
- **바로가기 HTML 타일**: Streamlit 버튼+컬럼으로는 네이버식 그리드가 안 나와서 순수 HTML로 재작성.

## 10. 업무 간소화 효과

- **취합 수작업 제거**: 10명 복붙(주당 30~60분) → 버튼 1회 HWPX. 미제출 자동표시·지난주 값 자동 채움.
- **양식 통일**(구매·공통확인), **이름 반복 입력 제거**(로그인=본인).
- **한 화면 파악**(공지·할 일·마감 카운트다운·일정·뉴스), **폰에서 앱처럼**.
- **속인화 제거**(담당자 없이 전원), **누락 방지**(만료 정리·마감 알림·문제 추적).
- **운영비 0원**, URL만으로 접속.

## 13. 운영 메모

- 인수인계 상세: 레포 루트 `AGENTS.md`. 만들어온 과정: `docs/dolbom_studio_소개.pptx`.
