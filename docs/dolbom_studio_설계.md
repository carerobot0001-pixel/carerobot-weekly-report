## 1. 한 줄 요약

**팀원이 각자 개인 계정으로 로그인해 쓰는, 돌봄로봇 사업단의 주간 업무·협업 웹앱.** 매주 반복되던
"10명 보고 → 한글 취합본 복붙"을 **버튼 한 번**으로 바꾸고, 구매·장비·방문·협업·공지·일정·뉴스를
한 화면에 모았다. 폰에서는 **홈 화면에 추가(PWA)** 로 앱처럼 쓴다.

## 2. 왜 시작했나 (문제 정의)

| 업무 | 기존 방식 | 문제 |
|------|-----------|------|
| 주간 업무보고 | 각자 한글/워드 작성 → 한 명이 **복붙 취합** | 취합자 매주 30~60분 수작업, 서식 깨짐, 미제출 파악 수동 |
| 공통확인사항 | 최혜민 연구원이 표 별도 관리 | 매주 한글 표 다시 제작 |
| 구매요청 | 엑셀 양식 개별 작성 | 양식 제각각, 수합 번거로움 |
| 장비 사용현황 | 엑셀 연구별 시트 수식 | 시트 동기화 부담 |
| 문제 접수/FAQ | 카톡·구두 | 기록·미해결 추적 안 됨 |
| 공지·일정 | 카톡 | 흘러가 놓침 |

→ **"입력은 각자, 취합·공유는 자동"** 하는 단일 웹앱으로 통합.

## 3. 목표 & 설계 원칙

- **취합 자동화 우선** — 사람이 하던 복붙을 없앤다(최대 페인포인트).
- **결과물은 기존 한글 양식 그대로**, 입력은 웹 폼으로 쉽게.
- **한 화면에서 파악** — 홈에 공지·할 일·일정·뉴스.
- **개인 계정 + 관리자 승인** — 누가 뭘 했는지 계정 기반(2026-07, 공용 비번 폐지).
- **데이터가 날아가지 않게** — 다층 백업(구글 복제 + 버전기록 + 휴지통 + 오프라인 백업).
- **폰 우선 접근성** — 반응형 + 홈 화면 추가(PWA).
- **점진적 확장** — 기능 1개 = 페이지 함수 + `*_store.py` 모듈.

## 5. 인증 — 개인 계정 로그인 (2026-07 전환)

공용 비밀번호(`carerobot`)를 폐지하고 **개인 계정 + 관리자 승인**으로 전환.

- 저장: 구글시트 `계정` 탭 — `[아이디, 비번(해시), 이름, 직함, 이메일_korea, 이메일_gmail, 상태, 가입일시, 승인일시]`
- **회원가입**: 이름·직함·아이디·비번(+확인)·이메일(korea·gmail) → 상태 `대기`.
- **관리자 승인**: `team_config.ADMIN_IDS`(현재 `{"admin"}`)에 있는 아이디로 가입하면 **자동 승인 + 승인 권한**.
  관리자는 사이드바 **👤 회원 관리**에서 전체 계정 상태(승인/대기/거부)를 변경(거부 복구·권한 회수 가능).
- **로그인**: 승인된 계정만. 로그인하면 `me`=이름, `is_admin` 세팅 → 모든 폼에 본인 이름 자동.
- **보안**: 비번은 **pbkdf2-sha256 해시**(평문 저장 X). 새로고침·자동 로그인은 **비번해시 기반 토큰**
  `?uid=&tok=`(위조 불가) + **브라우저 저장 자동 로그인**.
  보안 리뷰 통과(암호·토큰·로그아웃 정상, 거부 복구·행밀림 방지 반영).
- **자동 로그인 변천(§11-E 참고)**: ① 처음엔 `window.parent.localStorage` 직접 접근 → **iOS Safari에서
  막힘/앱 크래시** → ② revert → ③ **streamlit-js-eval**(대기 루프 없어 안 멈춤)로 재도입. 저장은
  `main()` 정상 렌더에서 1회(로그인 직후 rerun에 컴포넌트 쓰기가 잘리지 않게).

## 6. 기능 (사이드바 메뉴)

| 메뉴 | 하는 일 |
|------|---------|
| 🏠 홈 | 워드마크 → 바로가기 타일 → 공지 → **좌**(오늘 챙길 것·내 할 일) / **우**(그 외 일정·정보 미입력 일정) → 달력 → 뉴스 |
| 📝 업무보고 작성·취합 | (탭)작성 / 제출현황·**HWPX 취합본 생성**·백업 / 공통확인사항 |
| 🏠 스마트돌봄스페이스 | FAQ 수집 / 관리대장 문제 접수·완료 |
| 🛒 구매요청서 | 품목 표 → 시트 누적 + 엑셀 양식 + 완료처리 |
| 📋 문서 협업 | 구글문서 링크 보드 + 제출현황 + (OAuth)파일 자동변환 |
| 🔧 장비 사용현황 / 📍 실증 방문 일지 / 📚 과거 회의록 열람 | 조회·편집·기록 |
| 👤 회원 관리 (관리자) | 가입 승인·상태 변경 |

- **홈 바로가기**: 순수 HTML 타일 그리드(네이버식 아이콘). 클릭=`?go=` 링크(로그인 유지, 메뉴 이동/토글).
  공지등록·일정 관리(＋)·개인 할 일 추가(＋)는 같은 방식 토글(`?go=care/todo`, **명시적 open/close**).
- 사이드바 **기본 접힘**(폰/새로고침 시 다크 사이드바 방해 없게). 메뉴는 `»`로 연다.
- **개인 할 일**(`todo_store`, 2026-07-13): 오늘 챙길 것/내 할 일 옆 **＋** 로 **본인만 보는** 메모 추가·완료(삭제).
  캘린더 무관, 로그인 uid로 필터. `개인할일` 탭(구분=할일/챙길것).
- **일정 정보 미입력 알림**: 일정에 **장소** 칸 추가 + 14일 내 일정 중 **장소·시간 빈 것**을 홈에 표시(연가·조퇴 제외).
- **지난 보고 → 할 일 가져오기**(규칙 기반, AI 없음): 최근 주간보고의 **연구·업무 계획**을 줄 단위로 뽑아
  체크 후 내 할 일로 추가. (완료)·제목·[소제목] 노이즈 제외, 세부 줄(일시·장소 등)은 상위 항목에 묶음.

## 9. 핵심 설계 결정 & 근거

- **HWPX 취합본 = 셀 채우기**(동적 행 삽입은 한글 파서가 깨져서, 검증된 리프 표에 고정좌표). `TROUBLESHOOTING.md`에 과거 범인 기록.
- **구글시트 = 저장소**(서비스계정 R/W). 화재로 데이터 손실 경험 → 4중 안전.
- **드라이브 파일은 OAuth 본인계정**(서비스계정은 드라이브 저장 불가).
- **개인 계정 전환**(2026-07): 공용 비번 폐지, 계정+관리자 승인. 비번 해시·토큰.
- **문서협업 자동 공지**(가상 렌더, 완료·삭제 시 소멸) · **공지 만료 자동삭제**(RAW 저장으로 날짜 비교 안정).
- **바로가기 HTML 타일**: Streamlit 버튼+컬럼으로는 네이버식 그리드가 안 나와서 순수 HTML로 재작성.

## 11. 진행 과정 (변화 과정 — 누적 기록)

> "처음엔 이랬는데 → 이렇게 → 또 이렇게…" 실제 git 이력(160여 커밋) 기반. **막다른 길·되돌리기도
> 과정의 일부**라 그대로 남긴다. 최신 결과는 §1~10, 여기는 **어떻게 왔는지**.

### A. 취합 MVP (2026-04)
- 주간보고 웹 폼 → **버튼 1회로 HWPX(한글) 취합본** 자동 생성. 미제출 필드는 지난주 값 자동 채움.
- **최대 난관 — HWPX가 한글에서 안 열림**: 원인을 몰라 **이분탐색 디버깅**(Lv1~Lv7)으로 범인 추적.
  범인은 ① self-closing `hp:run` ② **ZIP mimetype 압축**(무압축이어야 함) ③ **flag_bits 바이너리** 등.
  → 안전장치로 **PDF·Word 대체 출력**도 추가(번들 한글 폰트로 깨짐 해결).
- 구글시트 API 쿼터 방지 캐싱, 과거 회의록 열람.

### B. 기능 확장 — 흩어진 업무 흡수 (2026-06)
- **스마트돌봄스페이스**(FAQ·관리대장 완료처리) → **구매요청서**(품목표→시트+엑셀) →
  **문서 협업**(구글문서 링크 보드 + **OAuth로 파일 자동 구글변환**) → **장비 사용현황** → 실증 방문일지.
- 패턴 정착: **기능 1개 = 페이지 함수 + `*_store.py` 모듈**. 엑셀·카톡 업무를 하나씩 앱으로.
- 중간에 OAuth 임포트 에러로 **앱 다운 → 복구** 경험(예외 처리 강화 계기).

### C. 홈 대시보드 & 담당자 제거 (2026-07-06~07)
- 메뉴가 8개로 늘어 **홈 대시보드 신설**(구글 캘린더 임베드·뉴스·공지·마감 카운트다운·내 할 일).
- 홈이 길어 **콤팩트화 + 오른쪽 레일 + 네이버식 HTML 타일 바로가기**(Streamlit 버튼 한계 탈피).
- **담당자(관리자) 개념 폐지 → 단일 비밀번호**(전원 동등), 담당자 대시보드 기능을 탭·토글로 분산.
- 공지 만료 자동삭제, 문서협업 자동 공지. 공통확인 탭을 업무보고로 이동.

### D. 브랜딩 & 개인 계정 (2026-07-09)
- **Dolbom Studio 브랜딩**: 주황 **DS** 아이콘(파비콘/홈화면)·**Dancing Script 워드마크**·주황/갈색 테마·다크 사이드바.
- **개인 계정 로그인 + 관리자 승인**으로 전환(공용 비번 폐지, 비번 pbkdf2 해시) → '나는 누구' 선택칸 제거,
  로그인=본인이라 폼에 이름 자동. 회원가입(이메일 korea·gmail·비번확인)·회원 관리 패널(승인/거부복구).
- **모바일 반응형 + 홈 화면에 추가(PWA)** 1차.

### E. 폰 안정화 & 자동 로그인 삽질기 (2026-07-09)
- 폰 실사용에서 문제 2건: ① 자동로그인 버튼이 **iOS Safari에서 실패** ② 달력 임베드 **구글 로그인 벽**.
- **자동 로그인 변천**: `window.parent.localStorage` 직접 접근 → **앱 크래시(대기 루프 멈춤)** → **revert** →
  **streamlit-js-eval**(멈춤 없음)로 **안전 재도입**. "실패해도 앱은 정상" 폴백 원칙 확립.

## 14. 로컬 AI 도입 방향 (설계 중, 미구현)

"주간보고 계획 → 진짜 할 일 스마트 추출", 요약 등 AI 기능을 **외부 API 없이 로컬**로.

- **판단 배경**
  - 규칙 기반 가져오기는 "제목 vs 하위 항목" 같은 **의미 판단**을 못 함 → 정확도 한계. AI가 필요.
  - **비용**만 보면 클라우드 API가 압도적으로 저렴(주 1회·소량 = 월 몇 백 원). 로컬의 값어치는 **"보고 내용 외부 미전송"**.
- **하드웨어(보유)**: 미니PC(ASUS PN54, 상시 ON) 1대 + 시놀로지 NAS + 구글드라이브 5TB.
  미니PC는 외장 GPU 없어 **CPU 추론** — **RAM = 안 터지냐, CPU = 빠르냐**를 가른다.
- **모델**: **EXAONE 2.4B**(LG, 한국어 강함, 소형이라 CPU에서도 수 초). 16GB로 **상주** 가능(~2GB).
  개발 중인 **유지관리 챗봇도 같은 2.4B** — 다만 챗봇의 최종 배포지는 **대상자 가정(엣지)** 이라 별개 트랙.
  당분간 사무실 미니PC = **홈페이지 관리 AI 전용**(챗봇은 가끔 사용성평가만). 2.4B라 **1대로 둘 다 돌려도 안 터짐**.
- **구조(예정)**
  ```
  어디서나/폰 → [홈페이지 앱(Streamlit Cloud)] ─(터널+토큰)→ [미니PC: FastAPI + EXAONE 2.4B 상주]
  ```
  - "인터넷 없이" = **모델이 외부 API를 안 부름**(PC는 네트워크 연결됨) → 클라우드 앱이 터널로 호출 가능.
  - 터널: **Tailscale Funnel**(도메인 불필요) 또는 Cloudflare(도메인 시). **토큰 인증** 필수(공개 앱).
  - 앱 쪽: `ai_store.py` + **"🤖 AI로 정리"** 버튼, **서버 꺼지면 규칙기반으로 자동 폴백**(앱은 정상).
- **속도/상주**: 상주(서버 시작 시 모델 로딩 유지) → 콜드스타트 제거. 대화형 챗봇일수록 상주가 유리.
- **상태**: 서버(미니PC+FastAPI+터널)가 서면 **① https 주소 ② 토큰 ③ 요청/응답 형식**만 확정해 앱에 연동.
  그 전까지는 **규칙 기반 가져오기로 운영**.
