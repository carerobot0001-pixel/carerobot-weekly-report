# dolbom studio — 설계 문서

> 돌봄로봇중개연구사업단 주간 업무·협업 웹앱
> 배포: https://carerobot-weekly-report.streamlit.app
> 작성: 김건양 연구원 (Claude Code / Codex 협업)

---

## 1. 한 줄 요약

**팀원 10명이 매주 흩어져서 하던 업무보고·취합·구매·장비·방문·협업·공지·일정을, 로그인 한 번으로 쓰는 웹앱 하나로 모았다.** 특히 "10명 보고 → 한글 취합본 만들기"라는 매주 반복되는 수작업을 **버튼 한 번**으로 바꿨다.

---

## 2. 왜 시작했나 (문제 정의)

기존 방식의 불편:

| 업무 | 기존 방식 | 문제 |
|------|-----------|------|
| 주간 업무보고 | 각자 한글/워드로 작성 → 한 명이 **복붙 취합** | 취합자 매주 30분~1시간 수작업, 서식 깨짐, 미제출자 파악 수동 |
| 사업단 공통확인사항 | 최혜민 연구원이 표 별도 관리 | 매주 한글 표 다시 만들기 |
| 구매요청 | 엑셀 양식 개별 작성 | 양식 배포·수합·양식 통일 안 됨 |
| 장비 사용현황 | 엑셀에 연구별 시트 수식 분류 | 시트 여러 개 동기화 |
| 문제 접수/FAQ | 카톡·구두 | 기록 산발, 미해결 추적 안 됨 |
| 공지·일정 | 카톡 | 흘러가서 놓침 |

→ **"입력은 각자, 취합·공유는 자동"** 하는 단일 웹앱으로 통합하기로 함.

---

## 3. 목표 & 설계 원칙

- **취합 자동화 우선**: 사람이 하던 복붙을 없앤다(가장 큰 페인포인트).
- **입력은 쉽게, 결과는 그대로**: 팀원은 웹 폼에 줄글만, 결과물은 기존 한글 양식 그대로.
- **한 화면에서 파악**: 홈에서 공지·할 일·일정·뉴스를 한눈에.
- **담당자 없이 전원 동등**: 특정인에 의존하지 않게(2026-07 단일화).
- **데이터가 날아가지 않게**: 다층 백업(구글 복제 + 버전기록 + 휴지통 + 오프라인 백업).
- **점진적 확장**: 기능 1개 = 페이지 함수 1개 + `*_store.py` 모듈 1개. 패턴 반복.

## 4. 아키텍처

```
┌────────────────────────────────────────────┐
│ Streamlit Cloud (streamlit.app)            │ ← 앱 호스팅(무료)
│   └ GitHub push → 자동 재배포               │
├────────────────────────────────────────────┤
│ Google Sheets "돌봄로봇_업무보고_제출함"     │ ← 모든 입력 데이터 저장
│   └ 서비스 계정(streamlit-bot@…)으로 R/W    │
│      (탭: 업무보고/구매요청/문서협업/장비현황│
│       /방문일지/공지/공통확인사항 …)         │
├────────────────────────────────────────────┤
│ Google Calendar (사업단 캘린더)             │ ← 일정(서비스계정 양방향)
│ Google News RSS                             │ ← 뉴스(키 불필요)
│ Google Drive (OAuth, 본인계정)              │ ← 문서협업 파일 자동변환
│ HWPX 템플릿(레포의 *.hwpx)                  │ ← 취합본/공통확인 생성용
└────────────────────────────────────────────┘
```

- **프론트+백 = Streamlit 단일 파이썬 앱**. 별도 서버·DB 없음(운영 부담 최소).
- **저장소 = 구글 시트**. 팀이 이미 쓰는 도구, 무료, 다지역 복제로 안전, 버전기록 내장.
- **비용 0원**: Streamlit 무료 티어 + 구글 무료 API 쿼터.

### 코드 구조
```
app/
├─ streamlit_app.py     ← 라우팅 + 모든 페이지 UI (홈/업무보고/스페이스/구매/협업/장비/방문/회의록)
├─ team_config.py       ← 팀원·이름목록·셀매핑·비밀번호
├─ sheets_store.py      ← 제출함 시트 R/W (업무보고)
├─ hwpx_exporter.py     ← HWPX 취합본 생성(셀 채우기, 매우 예민한 포맷)
├─ common_store.py      ← 공통확인사항 표→한글/엑셀
├─ purchase_store.py / collab_store.py / equip_store.py / visit_store.py
├─ space_store.py       ← 외부 시트(FAQ/관리대장)
├─ calendar_store.py / news_store.py / notice_store.py
└─ .streamlit/config.toml  ← 테마(주황/갈색), secrets.toml(비밀)
```

## 6. 핵심 설계 결정 & 근거

- **HWPX 취합본을 '셀 채우기'로 생성** — 동적 행 삽입은 한글 파서가 쉽게 깨져서, 검증된 리프 표에 고정 좌표로 값만 채움. `flag_bits`·자간(linesegarray)·탭문자 등 과거 5종 범인을 `TROUBLESHOOTING.md`에 기록.
- **구글시트 = 저장소** — 서비스 계정으로 R/W. "화재로 데이터 손실" 경험 때문에 **다지역 복제 + 버전기록 + 휴지통 + 오프라인 백업** 4중 안전.
- **드라이브 파일은 OAuth 본인계정** — 서비스 계정은 드라이브 저장 불가(quota 0). 그래서 문서협업 파일 자동변환은 본인 구글계정 OAuth(`drive.file`).
- **담당자 개념 제거(2026-07)** — 단일 비밀번호, `is_admin` 삭제. 취합·공지·백업·구매완료를 전원 개방(전체삭제만 확인 체크).
- **정체성 `me` 1회 설정** — 접속 후 이름 한 번 선택 → 세션+URL(`?me=`) 유지 → 모든 폼(작성자·요청자·방문자…) 기본값 자동.
- **문서협업 자동 공지(가상 렌더)** — 진행중 협업을 공지 영역에 계산해서 표시(시트 저장 없음) → 완료·삭제 시 자동 소멸.
- **공지 만료 자동삭제** — 표시 종료일 지나면 숨김 + 세션당 1회 정리. `RAW` 저장으로 날짜 문자열 비교 안정화.

## 7. 업무 간소화 효과

- **취합 수작업 제거**: 10명 복붙·서식정리(주당 ~30–60분) → **버튼 1회 HWPX 생성**. 미제출자 자동 표시, 미제출 필드는 지난주 값 자동 채움.
- **양식 통일**: 구매요청·공통확인이 항상 같은 엑셀/한글 양식으로 산출.
- **이름 반복 입력 제거**: `me` 1회 → 전 페이지 자동.
- **파악 시간 단축**: 홈 한 화면에 공지·할 일·마감 카운트다운·일정·뉴스.
- **속인화 제거**: 담당자 없이 누구나 취합·공지·백업.
- **누락 방지**: 공지 만료 자동정리, 협업 마감 임박 알림, 문제 미해결 추적.
- **운영비 0원**, 별도 설치 없이 **URL + 비밀번호**만으로 접속.
